### Some time ago...

I was really overwhelmed by the enormous world of JavaScript. I was taken aback the moment I dipped my foot in *it*.

Can you imagine? For almost two years, I was in denial — telling myself I didn't need to learn JavaScript or at least try writing scripts with it. In my defense, I finished my thesis and graduated college by relying on how ASP.NET C# passed data to the template, and just using the default JavaScript files from templates, boilerplates, and frontend frameworks — **as is**.

Lucky me: the first task at my first job was to build a WEB API. Everything was server-side, so I didn't have to touch frontend development *(I slipped through the inevitable again)*. But after finishing that project, I was asked to create a web tool — with dashboards, stats, etc. For a moment, I thought I could get away without writing any JavaScript. I thought Django templates would be enough. I was wrong. I wasn't ready. But I had a task to finish. From that moment, I thought: "You need to learn JavaScript. You *need* it."

### But before that...

I did what some of you did too. I searched for the very best JS framework — like no one ever was.

![I wanna be the very best]*(broken image — was on zorexsalvo.com)*

Based on that search, the top JavaScript frameworks for 2017 Q2 were... *drum roll*... **Angular**, **React**, **Ember**, **Meteor**, **Vue**, and many more. Everyone has their own top-tier frameworks. For me, I just needed to get started. So I picked the easiest one that would get me hooked quickly. Reading through blogs and articles, I kept seeing two keywords: **EASY** and **SIMPLE**. So you probably already know why **Vue.js, I choose you!**.

### Easy and Simple

For me as a beginner, those words were like a magnet. I chose it right away.

![Choices]*(broken image — was on zorexsalvo.com)*

The first thing I did with Vue was create a Todo app. I found a step-by-step video tutorial by Academind on YouTube. It was fun. I didn't imagine frontend coding could be this fun and easy. But honestly — with that <1 hour tutorial, there was so much magic happening, and I was clueless how Vue did any of it. I didn't learn much, but my takeaway was experiencing how awesome Vue felt.

Since I was hyped, I kept studying and experimenting. I started from the beginning: scripts on JSFiddle, learning directives, DOM interaction, list rendering, and conditionals. After the basics, I created a simple CRUD app called **Ideya** — a repository for my ideas. [My blog](http://zorexsalvo.com) had bits of Vue in it too.

![Ideya]*(broken image — was on zorexsalvo.com)*

I was really proud of myself after finishing that project. But I wasn't satisfied yet. Skimming through the documentation, I read about vue-cli, webpack, components, and more. I'm not stopping here. There's still so much to explore.

> **Note 2026:** A lot has changed since 2017! Vue is now at version 3 with the Composition API, Vite has replaced webpack as the build tool of choice, and the ecosystem has matured significantly. But the core appeal — easy to learn, gentle learning curve — still holds true.
