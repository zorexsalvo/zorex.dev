Let's start with the text editors I've used since the beginning of my programming journey.

The very first language I learned was C, in my first year of college. I don't remember the name of the editor. It was like a command prompt with neon blue background, gray menu and status bars, and bright green font. I don't think it was usable for languages other than C.

In my second year, we had Cobol, Java, and C# on our curriculum. For Cobol, we used Notepad. For Java, we used Notepad++ because plain Notepad was really lacking — no undo, no syntax highlighting, etc. Then MS Visual Studio came along — a very powerful IDE, so powerful it drove my laptop (3GB RAM + Core 2 Duo) to its limit every damn time. But I thought Visual Studio was a gift from heaven with its full-packed functionality — drag and drop, intellisense, graphical interfaces, compiler, debugging, and many other tools. I used it from second year through fourth year. I think it was worth sacrificing my laptop to that resource-hungry IDE for easier development.

**Internship.** I was accepted at a startup. Startups usually don't use proprietary software because of expensive licenses, so Microsoft technologies were a no-go. The first thing we did was install an Ubuntu-based Linux distro — Lubuntu. Linux meant goodbye to my favorite, Visual Studio *(sobs)*.

During my internship, I used Sublime 2. It was good and lighter than Visual Studio. It took a while to adapt to this new editor without the fancy functionalities. But shifting editors was beneficial — my laptop was already dying after almost 7 years, and not relying on intellisense actually made me a better programmer.

![VSCode]*(broken image — was on zorexsalvo.com)*

**Being a professional.** I continued with Sublime for a while, but then Visual Studio Code came out — open-source. I quickly switched, and I never went back. VSCode had promising features: integrated terminal, Git support, plugins.

![Vim]*(broken image — was on zorexsalvo.com)*

I was happily using VSCode until I developed a strain in my mouse hand. I searched for alternatives to minimize mouse use, and discovered Vim. Before Vim, I wasn't practicing proper typing, which contributed to the strain. Learning Vim forced me to use home keys because the navigation (h, j, k, l), insert mode, visual mode — everything relies on the keyboard. Those commands are really hard to press if you don't know how to place your fingers correctly.

What I love about Vim: it's lightweight, runs in the terminal, and is highly configurable. You can make it work like any other editor thanks to the vast open-source plugin ecosystem. Here's my minimal Vim config: [vimrc](https://github.com/zorexsalvo/vimrc).

Despite the steep learning curve, I'm not planning to leave Vim anytime soon. This editor and its plugins are a whole 'nother journey.
